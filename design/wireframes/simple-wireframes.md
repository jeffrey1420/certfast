# CertFast - Simple Wireframes

## Screen 1: Landing Page

```
+------------------+
| Logo    Nav      |
+------------------+
|                  |
|   HEADLINE       |
|   subtext        |
|   [CTA Button]   |
|                  |
+------------------+
| Feature 1 | 2 | 3|
+------------------+
|     Footer       |
+------------------+
```

### Layout Description

The landing page wireframe follows a clean, conversion-focused design pattern that guides visitors toward the primary call-to-action. The header section establishes brand identity with the logo positioned on the left, creating immediate visual recognition. The navigation elements reside on the right side of the header, providing quick access to key sections such as features, pricing, and documentation. This balanced layout ensures users can navigate freely without losing sight of the brand.

The hero section dominates the center of the viewport, featuring a bold headline that communicates the core value proposition in a single, impactful statement. Below the headline, subtext provides additional context and supporting details that help visitors understand what CertFast offers and why it matters to them. The call-to-action button stands out prominently beneath the text, designed with high contrast to draw attention and encourage clicks. This button represents the primary conversion goal, whether that's starting a free trial, scheduling a demo, or signing up for the platform.

Beneath the hero section, three feature cards are arranged horizontally to showcase the platform's key capabilities. Each card contains an icon, title, and brief description highlighting different aspects of CertFast's compliance management features. This layout allows for quick scanning and helps visitors grasp the platform's value at a glance. The cards are evenly spaced to create visual balance and professional presentation. The footer section anchors the page with essential links, copyright information, and additional navigation options, completing the standard landing page structure while providing utility for users who scroll to the bottom.

## Screen 2: Dashboard

```
+--------------------------------+
| Sidebar | Header               |
|         |----------------------|
| Nav     | Metrics Grid         |
| Links   | [Score][Assessments] |
|         | [Evidence][Days]     |
|         |----------------------|
|         | Progress Bar         |
|         | 75% Complete         |
+--------------------------------+
```

### Dashboard Description

The dashboard wireframe presents a comprehensive overview interface designed for quick status assessment and navigation. The left sidebar serves as the primary navigation hub, housing links to different sections of the application including Assessments, Evidence Library, Reports, Settings, and User Management. This persistent navigation element remains visible at all times, allowing users to jump between different functional areas without losing context or returning to a main menu. The sidebar's vertical layout maximizes screen real estate while keeping all navigation options within easy reach.

The top header bar spans the full width of the content area and contains contextual information such as the current page title, user profile dropdown, notifications bell, and search functionality. This header provides immediate access to user-specific actions and system-wide search capabilities, ensuring users can find information quickly regardless of their current location within the application.

The main content area is dominated by a metrics grid displaying four key performance indicators critical for compliance monitoring. The Compliance Score card shows the overall certification readiness percentage, calculated from completed requirements across all assessments. The Assessments card displays the total number of active and completed assessments, helping users track their workload. The Evidence card counts the pieces of uploaded documentation supporting various compliance requirements. The Days to Audit card provides a countdown timer showing how much time remains before the next scheduled audit, creating urgency and helping teams prioritize their efforts. Below the metrics grid, a prominent progress bar visualizes overall certification completion status, currently showing 75% complete with a clear label indicating the current milestone achievement level.

## Screen 3: Assessment Detail

```
+--------------------------------+
| < Back  Assessment Name   Edit |
+--------------------------------+
| Status: [In Progress]          |
+--------------------------------+
| Evidence Upload Zone           |
| [Drop files here]              |
+--------------------------------+
| Checklist:                     |
| [x] Req 1 - Done               |
| [ ] Req 2 - Pending            |
| [ ] Req 3 - Pending            |
+--------------------------------+
```

### Assessment Detail Description

The assessment detail view provides a focused workspace for managing individual compliance assessments. The top navigation bar establishes context and control, featuring a back button on the left that returns users to the assessments list without losing their place in the workflow. The center of the header displays the assessment name prominently, ensuring users always know which assessment they're working on. An edit button on the right enables quick access to modify assessment details such as name, description, assigned team members, and due dates.

Directly beneath the header, a status badge clearly indicates the current state of the assessment. In this wireframe, the status shows "In Progress" but the badge system supports multiple states including Not Started, In Progress, Under Review, and Complete. The visual styling of the status badge uses color coding to communicate state at a glance, helping teams quickly identify which assessments need attention.

The evidence upload zone occupies a significant portion of the interface, providing a dedicated area for document management. This zone supports drag-and-drop functionality allowing users to quickly upload supporting documentation from their local file system. The drop zone displays clear instructions and visual feedback during file operations, ensuring users understand how to add evidence to the assessment. Below the upload area, a requirements checklist displays all compliance requirements associated with this assessment. Each checklist item includes a checkbox indicating completion status, with checkmarks showing finished requirements and empty boxes indicating pending work. The list displays requirement names alongside their status, creating a clear action-oriented view that helps users track progress and identify remaining tasks. This checklist serves as the primary interaction point for marking requirements complete as teams gather and submit supporting evidence.
